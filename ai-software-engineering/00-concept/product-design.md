# AnyDownloader — 产品设计

## 产品定位

面向桌面浏览器（及后续可扩展 Electron）的**通用下载管理**工具：用户粘贴或输入资源地址，系统**自动识别协议**并进入对应下载管线；主界面以**下载列表**为中心，右侧 **Dock** 展示详情、测速与分块进度可视化。

## 壳层类型

- **SDI（单文档界面）**：单一主窗口（单页应用视口）；多任务以列表行表示，不采用 MDI 子窗口。
- **Web / 页内仿壳**：**菜单栏**与**标题条**为页内 DOM **纵向两行**（先标题条，后菜单栏），不将标题与顶层菜单混排在同一行。

## 纵向分区（页内）

| 区域 | 职责 |
|------|------|
| 标题条 | 应用图标/名称、可选脏标记（本阶段无编辑文档，可省略） |
| 菜单栏 | 文件、编辑、**语言**、**主题**、窗口（Dock）、**帮助**（页内纵向第二行，与标题条分行） |
| 工具栏 | 添加任务、开始、暂停、删除等；图标为主，组间分隔 |
| 客户区 | 中部下载列表 + 右侧 **Dock Area**（下载信息 / 网络测速 / 下载块图） |
| 状态栏 | 就绪/进度摘要；**点击打开 Log 窗口**（纯文本、可复制全文） |

## 右侧 Dock Area

- **Dock Area** 含 **Dock View**（内容）与 **Dock Button Bar**（贴窗口**右缘**的折叠按钮条）。
- 阅读顺序：**客户区 → Dock View → Dock Button Bar**。
- Dock：**下载信息**、**网络测速**、**下载块图**；可折叠/展开，可从菜单「窗口」恢复。

## 协议与能力边界（浏览器 MVP）

| 协议 | 识别 | 当前实现阶段 |
|------|------|----------------|
| `http:` / `https:` | 是 | 使用 `fetch` 流式拉取并保存（受 CORS/混合内容限制） |
| `blob:` | 是 | 同源 `Blob URL` 可走 `fetch` 流式保存 |
| `ftp:` / `ftps:` / `sftp:` | 是 | 需本机代理或桌面壳，UI 可排队并提示 |
| `magnet:` | 是 | 需 WebTorrent/后端，UI 可排队并提示 |
| `ed2k:` / `thunder:` 等 | 是 | 识别为专用链；需专用引擎或桌面壳，UI 可排队并提示 |
| `data:` | 是 | 可内联解码为小文件保存 |
| `ws` / `wss` / `file` / `ipfs` / `rtsp` / `mms` / `s3:` / `git:` / `dav` / `davs` 等 | 是 | 识别并列表展示；浏览器内下载管线占位，需后续扩展 |

**自动识别**：由前端根据 URL scheme 与模式匹配选择处理器；无需用户手选协议。

## 非目标（初版）

- 不在浏览器内实现完整 BitTorrent/FTP 协议栈（架构预留扩展点）。
- 不替代企业合规审计；第三方许可见仓库根 `THIRD_PARTY_LICENSES.md`。
